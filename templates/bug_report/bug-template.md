## 严重度判断 / Severity Guide

| 级别 | 定义 | 例子 |
|---|---|---|
| **P0** | 全产品停摆 / 数据丢失 / 安全事故 / 资损 | 主页面 500 / 客户数据被互相看到 / 财务计算错了几十个客户 |
| **P1** | 核心工作流阻断 / 重要客户被影响 | 客户上传视频功能失败 / 大客户的工作流断 |
| **P2** | 边角功能 / 小范围影响 / 有绕行 | 某个少用 Tab 的搜索框出错 / 一类客户能用其他方式 |
| **P3** | 体验性问题 / 文档错别字 / 无真实影响 | UI 按钮颜色不对 / 文档错别字 |

不确定时**往严重判**。降级比升级容易。

---

## P0 / P1 的特殊纪律

P0 / P1 不是"登记完就行"——立刻：
- 通知工程负责人
- 通知受影响业务方（销售 / 客户成功 / 客服）
- 如客户已感知 → 客户告知（按红线 #2 客户面口径）

不要"先登记，明天再说"。
