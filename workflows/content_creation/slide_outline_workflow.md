# PPT / 演示大纲 / Slide / Presentation Outline

> 适用：销售、产品、运营、管理在做客户演示、内部分享、对外路演时。
> For: sales / product / ops / management preparing customer demos, internal shares, external pitches.

---

## 一句话 / One Line

**PPT 不是给你看的 cue card，是给观众看的论证。** AI 帮你做大纲，**你**拍板每一页"听众读完后应该想什么 / 做什么"。
**Slides aren't your cue card; they're the audience's argument.** AI drafts outlines; **you** decide what each slide makes the audience think / do.

---

## 4 步法 / 4-Step Process

### 步 1：定"演示完后听众想什么 / 做什么" / Step 1: Define the Outcome

不是"我要讲一下我们的产品"，是：
Not "I'll talk about our product", but:

> "演示完成后，希望听众 (a) 理解我们 vs 对手的差异、(b) 同意安排下周二的实施 kickoff 会、(c) 把我们的方案推荐给他们的法务 / IT 团队。"

```
我要做一份演示给 [听众]，时长 [X 分钟]。

演示完成后我希望他们：
1. 理解：[2-3 件事]
2. 同意 / 决定：[1-2 件事]
3. 行动：[1 件事，下一步具体动作]

请帮我从这 3 个目标倒推大纲：
- 目标 1 需要哪些 slides？
- 目标 2 需要哪些 slides？
- 目标 3 需要哪些 slides？
- 总共 N 张 slide，每张 ~M 分钟
```

### 步 2：起每页大纲 / Step 2: Slide-by-Slide Outline

```
基于上面的目标-slide 倒推，请详细到每一页：

第 N 页：
- 标题 / Title: [一句话，让没看正文的观众也能 get 这页的结论]
- 关键信息 / Key info: 2-4 个 bullet
- 视觉建议 / Visual: 是否需要图 / 图表 / 截图 / 流程图
- 演讲者要补充的 / Speaker note: 这页除了 slide 上的内容，演讲者要口头说什么

约束：
- 每页只解决一件事；如果一页 cover 多件事，拆成多页
- 标题必须是结论性的（"我们的产品有 3 大优势" 而不是"产品介绍"）
- 不出现客户名 / PRD 编号 / 内部代号（红线 #2）
- 数据必须有依据（注明出处）
```

### 步 3：30 秒测试 / Step 3: 30-Second Test

```
对每一页 slide，回答：

1. 如果听众只看 30 秒就走了，他们能 get 什么？
2. 这一页的标题独立读起来，还能传达页面核心信息吗？
3. 这一页可以删掉吗？删了之后整个演讲少了什么？
   - 如果"少了点过渡感"——那就是装饰页，删 / 合并
   - 如果"少了关键论据"——保留
```

不能通过 30 秒测试的 slide，要么改标题，要么删。
Slides that fail the 30-second test → improve title or delete.

### 步 4：起讲稿 / Step 4: Talk Track

```
基于最终大纲，写讲稿：
- 每页 ~M 分钟（按上面的总时长倒推）
- 用第一人称 + 自然口语
- 标出"这里我会停下问 [谁] 一个问题：[问题]"
- 标出"这里如果时间紧可以跳过"

最后给我一份"应急 5 分钟版"——如果只有 5 分钟，最关键的 X 张 slide 是哪几张。
```

## 客户演示 vs 内部分享的差异 / Customer Demo vs Internal Share

### 客户演示

- **不点名其他客户**（除非客户公开过 + 法务过审）/ Don't name other customers
- **不暴露内部代号 / 排期**（红线 #2）/ Don't expose internal codes / roadmap
- **承诺要可兑现**——演示中口头说的承诺等同于书面 / Verbal promises in demos = written
- **下一步要具体**——演示完后的具体动作有时间表 / Next steps with timelines

### 内部分享

- 可以提具体客户名 / 内部代号（在内部范围）/ Can name customers / use internal codes
- 可以更深入数据 / 财务 / Can go deeper on data / finance
- 但**不要假设"内部"= 信息无敏感**——会议录音 / 截屏可能流出 / "Internal" doesn't mean "leak-proof"

## 演示前的"反方角色" 演练 / Pre-Demo "Devil's Advocate"

```
基于上面的大纲，请假装你是 [听众类型，比如"挑剔的潜在客户"]，列出 top-5 你会问的难题：

每个难题：
1. 问题描述（具体、不留余地）
2. 我应该如何回应（含数据 / 案例 / 退路）
3. 如果我答不好，对方会怎么继续追问
```

演示前自己练 1-2 遍这些"难题"。
Practice these tough questions 1-2× before the demo.

## AI 起 PPT 的常见错误 / Common AI Slide Errors

| 错误 / Error | 表现 / Symptom | 修正 / Fix |
|---|---|---|
| **每页太满** | 一页 8 个 bullet | 一页 ≤ 4 个 bullet；超出拆页 |
| **标题不结论化** | "产品介绍" | 改成结论："我们的产品有 3 大优势" |
| **画面建议都是"插入图"** | 每页都"建议加一张图" | 标具体的图：截图？流程图？图表？ |
| **讲稿照念 slide** | 讲稿和 slide 内容一字不差 | 讲稿是"补充" slide，不是重复 |
| **过渡页没用** | "下面我们来讲 X" 占一整页 | 删掉，让 X 页直接接上 |
| **结尾"谢谢"页** | 一整页大写"谢谢" | 改成"下一步行动：[具体]" |

## 双语演示 / Bilingual Slides

跨国 / 跨地域客户的演示常需要中英对照。
Multinational / cross-region demos often need CN+EN.

不要让 AI 直接翻译。让懂目标语言的人**重写**（而不是翻）。
Don't AI-translate. Have a native speaker **rewrite** (not translate).

每页双语布局选一种保持一致：
Pick one bilingual layout and keep consistent:

| 布局 / Layout | 适合 / Best for |
|---|---|
| 中文为主 + 英文小字注 | 国内为主、外籍辅助 |
| 中英并排（左中右英） | 双语听众均衡 |
| 中英分两套（中文版 + 英文版） | 听众分两批 |
| 仅一种语言 + 实时口译 | 现场有口译员 |

## 速查 / Cheat Sheet

```
1. 定演示完后听众想 / 做什么（不是"讲一下产品"）
2. 倒推大纲（从目标到 slide）
3. 30 秒测试（每页能 30 秒 get 吗？）
4. 起讲稿 + 应急 5 分钟版

陷阱：每页太满 / 标题不结论化 / 讲稿照念 slide / 过渡页 / "谢谢"页
红线：不点名客户 / 不暴露内部 / 承诺可兑现 / 下一步具体
```
