## 共同纪律 / Shared Discipline

- 红线 #5：改代码前必有 PRD；而 PRD 前如果需求没挖清，先开 meeting（PRD 是合同，meeting 是收敛输入）
- 红线 #6：非平凡决策落字成 ADR
- 红线 #10：完成前过五件套清单
- 决策跑偏时回到至高决策原则（长期 / 业界 / 核心目标）
