## §1 你是谁 / 今天可能在做什么 / Who You Are & Today's Likely Work

你可能是：实习生 / 刚入职跨职能岗位 / 从其他行业转入 / 项目经理（同时碰多角色）/ 临时被借调到这个团队。本周可能在做：

1. 翻仓库找"我这件事用哪份工作流"
2. 第一次写 PRD / 跟着别人写过的 PRD 学
3. 帮其他角色出半成品（销售简报 / 视频脚本 / 数据小报）
4. 周会前整理上下文
5. 听不懂某个术语 → 想问 AI 但不知道怎么开口

## §2 第一次和 AI 的对话 / Your First AI Conversation

把下面整段复制给 AI（替换 `<...>` 处）：

```
请先读 AI_MANUAL.md。我对仓库还陌生，今天我想做 <一句话描述任务>。我不确定该用哪份工作流。请：1) 对照 AI_MANUAL §4 任务-入口表给我 1-2 个最匹配的工作流，2) 在我动手前指出该工作流会触发哪些红线（特别是 #2 #3 #5），3) 不要给我"看起来合理"的虚构路径——不存在就直说。

Read AI_MANUAL.md first. I'm new here; today I want to do <one-line task>. I'm not sure which workflow fits. Please: 1) match it to AI_MANUAL §4 task-entry table, 2) flag which red lines (esp. #2/#3/#5) will be triggered, 3) don't fabricate plausible-looking paths — say "doesn't exist" if so.
```

## §4 三个新手最常踩的坑 / 3 Common Pitfalls

1. **想"读完整个仓库再开工"**：仓库 25+ 个一级文件、9 个目录、数百份 Markdown——读完要好几天。**正确做法**：找你今天的任务最近的 1-2 份工作流就开干，剩下随用随查。
2. **猜自己角色对应哪份**：猜错比不挑更糟。**正确做法**：用本份的 §2 脚本让 AI 帮你对照 [`README.md`](README.md) 里的 14 角色路径表挑。
3. **不敢问真人**：跨职能岗位最大风险是"AI 帮你产出了一份团队约定俗成不该这样写的东西"。**正确做法**：第一次产出**强制**找 1 个该业务线同事 spot-check，不要直接发出去。
