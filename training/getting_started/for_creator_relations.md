## §1 你是谁 / 今天可能在做什么 / Who You Are & Today's Likely Work

你是 TUZHAN 短视频 AI Agent 团队的创作者运营。对接对象是外部达人 / KOL / MCN（不是 B2B 客户、不是自家账号矩阵）。本周高频场景：

1. 达人筛选 / 数据评估
2. 合作邀请邮件 / DM
3. 合作协议沟通（先和法务核合同模板）
4. 达人内容审稿（合规 + 调性）
5. 分账核对 / 阶段性结算

---

## §2 第一次和 AI 的对话 / Your First AI Conversation

把下面整段复制给 AI（替换 `<...>` 处）：

```
请先读 AI_MANUAL.md。我是创作者运营，今天要起 <达人类型 X> 的合作邀请邮件 + 1 页内容审稿 checklist。达人姓名 / 真实合同金额 / 内部代号都已脱敏。涉及合同条款的部分**只引用法务给的模板编号**，不要 AI 自创条款。先列你不确定的点。

Read AI_MANUAL.md first. I'm in creator relations. Today: draft an outreach email + a 1-page content-review checklist for <creator type X>. Creator names / amounts / codenames redacted. For contract clauses, **only reference legal-provided template IDs** — no AI-invented terms. List unknowns first.
```

## §4 三个新手最常踩的坑 / 3 Common Pitfalls

1. **达人合同金额 / 分账明细贴 AI**：越红线 #3。脱敏后再用，规则见 [`principles/subs/confidentiality.md`](../../principles/subs/confidentiality.md)。
2. **外部达人邮件含内部产品代号 / 模型 ID / 内部岗位代号**：越红线 #2。达人会转发到 MCN 群——内部代号一旦出去就回不来。
3. **没看法务模板就让 AI 自创合同条款**：分账 / 独家 / 用途授权这些条款必须按法务给的模板写，AI 自由发挥 = 公司可能背上一份不平等合同。
