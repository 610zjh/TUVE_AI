## 1. 背景与动机

销售反馈：电话后整理跟进材料是高频低价值动作，每次 30-60 分钟。
按 [`workflows/customer_communication/sales_call_followup.md`](../../../../workflows/customer_communication/sales_call_followup.md) 流程，整理产出 3 类内容（客户面简报 / 内部简报 / CRM 更新）。

我们做了一份小型内部工具，让 AI 起草前两类，把销售时间从 30-60 分钟降到 5-10 分钟。

不做这件事的代价：
- 销售一周累计在跟进整理上 ~3-5 小时
- 跟进延误（24 小时内不发邮件）会让回复率下降
- AI 起草的版本如果不规范，会引入红线 #2 / #3 风险

---

## 2. 目标与非目标

### 目标
- ✅ 销售从原始电话笔记到两类简报草稿的时间从 ~45 分钟降到 ≤ 10 分钟
- ✅ 输出对红线 #2（客户面文案）+ 红线 #3（PII）有自动检查
- ✅ 工具支持"无 API key 也能跑"（让任何销售都能体验）

### 非目标
- ❌ 不做"自动发送邮件" —— 红线 #8 不可逆动作必须人确认
- ❌ 不做"客户名单 / CRM 直连" —— 红线 #3 数据脱敏先在销售自己端做
- ❌ 不做 GUI / 网页版（v1）—— CLI 足够；GUI 进 v2 候选

---

## 3. 用户故事

- 作为销售，我希望电话结束后 5 分钟内有一份简报草稿，让我能 24 小时内完成跟进发送
- 作为销售总，我希望简报草稿里的"承诺给客户的事 / 客户答应的事"能被结构化成可跟踪的 task
- 作为合规 / 法务，我希望工具会自动告警含有"PII 未脱敏" / "客户面含内部代号" 的输入 / 输出

---

## 4. 需求详述

### 4.1 功能需求
- 4.1.1 接受文本文件输入（电话笔记）
- 4.1.2 提取客户标签 + 电话日期
- 4.1.3 调 AI 生成两类简报：external（给客户）+ internal（给团队）
- 4.1.4 红线 #2 自查：扫输出中的内部代号 / PRD 编号 / Bug 编号 / 模型 ID 等
- 4.1.5 红线 #3 自查：扫输入中的真实手机 / 邮箱 / 身份证（如发现，警告并退出）
- 4.1.6 输出到本地文件（不发送、不上传外部）

### 4.2 非功能需求
- 性能：本地脚本，AI 调用约 10-30 秒（取决于网络）
- 合规：不存任何输入 / 输出到我方任何外部服务
- 可观测性：所有警告写到 stderr
- 回退方案：无 API key 时进入"模板填充模式"，仍能完成基本流程

## 8. 风险与对策

### 风险 1：销售把未脱敏的 PII 直接喂给 AI
- 概率：高
- 影响：高（红线 #3 违规）
- 对策：脚本启动时先扫 PII，发现就警告并退出（不会送给 AI）
- 触发条件：默认每次都执行此检查

### 风险 2：AI 输出的"客户面简报"含内部代号
- 概率：中
- 影响：中
- 对策：自查 + 警告（仍写入文件，但销售必须看警告再决定是否发）
- 触发条件：每次输出后扫描

### 风险 3：销售把 AI 草稿当终稿直接发
- 概率：中
- 影响：中（草稿可能不够精确）
- 对策：脚本输出后明确提示"下一步：人审 + 通过红线 #2 三道闸"
