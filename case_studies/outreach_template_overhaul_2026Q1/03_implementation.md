# 03. 5 周落地

---

## 时间线

| 周 | 关键动作 |
|---|---|
| W1 | 起 PRD-0051 + ADR-0028 + ADR-0029；销售 / 营销 / 视频 / 数据各自任命 owner |
| W2 | 营销起草 3 套模板 v1；视频开始拍 3 类素材 |
| W3 | 数据建 A/B 测试基础设施（追踪打开 / 回复 / 转 demo） |
| W4 | A/B 测试启动（2 周）|
| W5 | A/B 数据出 + 选最终版本 + 上线 |

---

## A/B 测试纪律 / A/B Test Discipline

### 设计 / Design

每套模板（决策者 / 影响者 / 执行者）测：
- 变量：开头 5 句话的语气（数据型 vs 故事型 vs 痛点型）
- 控制变量：发送时间 / 收件人样本 / 视频素材
- 样本量：每变体至少 200 条（这样 95% 置信区间能区分 2% 和 3%）
- 测试时长：2 周

### 数据收集

数据团队搭了一个简单的看板（不是大型 BI 工具——红线 #15 主次审视：先做扎实简单的）：
- 打开率 / 回复率 / 24h 内回复率 / 转 demo 率
- 按变体分

### 决策标准

预先约定：
- 如果某变体回复率显著高于其他（统计上 p < 0.05），用它
- 如果差异不显著，看次级指标（24h 回复率、转 demo 率）
- 如果还是不显著，选"语气最贴近 TUZHAN 品牌"的——主观判断由营销总拍板

## 第一版"营销腔"过重的失误 / Marketing-Speak Mistake in v1

### 发生

W4 中段，A/B 测试启动后 5 天，销售反馈：
- "客户回复'你们这听起来像 SEO 广告'"
- "好几个客户直接没回，连不感兴趣的礼貌回应都没有"

我们立刻看了 v1 的内容：

```
（决策者版本，v1 开头）

XX 总您好！

🚀 在数字化转型时代，TUZHAN 致力于通过 AI 驱动的革命性平台，
为像您这样的行业领袖提供全方位的内容产出解决方案。
我们已经为众多客户实现了显著的效率提升 ...
```

红线 #2 + 营销文案禁忌全中：
- 形容词堆砌（"革命性""数字化转型""全方位""显著"）
- AI 套话（"在 X 时代"）
- 抽象的客户故事（"众多客户"）

不是技术上违反红线 #2 的"内部信息"，但**违反了 [`content_quality.md`](../../principles/subs/content_quality.md) 的"内容质量"原则**。

### 紧急修复（W4 第二周）

ADR 没改，但模板 v1 → v2：

```
（决策者版本，v2 开头）

XX 总您好。

我们看到您所在的行业（[具体行业]）短视频投入持续加码，
但同时遇到产能瓶颈——这是我们最近 6 个月遇到 9 位类似客户的反馈。
能否占用您 5 分钟，分享我们如何帮一家同行业客户从每周 50 条产能升到 500 条？
```

差异：
- 直接给客户具体数据（同行业 9 位客户的反馈）
- 不堆形容词
- 直接给"想要的下一步"（5 分钟分享）

### A/B 测试结果

v2 上线后剩余 9 天的 A/B 数据：
- 决策者层 v2 的回复率：**4.7%**（v1 仅 1.5%）
- 影响者层 v2：**3.8%**
- 执行者层 v2：**4.1%**

→ v2 大幅胜出 v1。决定 v2 作为最终版本。

---

## 整体上线后数据 / Post-Launch Overall Data

5 周后（上线 + 4 周 A/B + 整体上线后 2 周）的数据：

- 总外联数：920（这 6 周）
- 回复率：**4.2%** ✅ 远超 2.5% 目标，达到业界 top quartile
- 转 demo 率：**18%** ✅ 优于 15% 目标
- 平均回复时长：**1.4 天** ✅ 优于 1.8 天目标

---

## 教学点 / Teaching Points

- **A/B 设计纪律**：预先约定决策标准（统计显著性 + 次级指标 + 主观兜底），不要 A/B 出结果后才"挑对自己有利的"
- **样本量先算**：要测出 1.8% vs 2.5% 的差异需要每变体 ~200 条——预先算清楚再开始
- **快速纠正**：v1 上线 5 天就发现"营销腔"问题——靠销售在前线的真实反馈
- **AI 起的"营销文案"容易堆形容词** —— 必过 [`content_quality.md`](../../principles/subs/content_quality.md) 的反陷阱清单
- **数据团队不必上来就建大型 BI**：简单的 A/B 看板就够；先扎实再升级
