## 做对的 / What Went Right

### 1. W1 主动提了 7 个澄清问题再开工

工程 lead 读完 PRD 后没有立刻开干，而是按 [`prd_to_implementation.md`](../../workflows/engineering/prd_to_implementation.md) §"开工前" 列了 7 个不确定点：

1. AC-3 的"客户数据规模" 具体是多大？最大客户的什么量级？
2. AC-1 的"7 个核心数字" 是这 7 个还是可调？产品说"可调"我们就不能写死
3. AC-2 的"详细看板" 已存在还是新建？
4. 移动端要不要适配现有的设计系统？
5. 新员工 onboarding 文档由谁写？产品 / 客户成功？
6. 红线 #2 三道闸由谁过？工程过 grep + 品牌声音由谁过？
7. 第二阶段的"客户 beta" 是哪些客户？要预先告知吗？

产品花了一天补完，开工前所有不确定都消失。这一天**省下了后续 1-2 周的返工**。

### 2. W2 提前做了负载测试

按 §8 风险 1 的对策，第 2 周（不是上线前）做了"最大客户数据规模"下的负载测试。结果：

- P95 = 1100ms ❌ 不达标 800ms
- 根因：4 个核心数字的查询是分别打数据库的；大客户数据下数据库 IO 是瓶颈

立刻起 ADR-0027（数据预聚合方案）：每 5 分钟生成一份核心数字的预聚合表，首页直接读这份表 → P95 降到 ~200ms。

如果不做提前负载测试，按原计划写完所有代码后才发现 → 上线倒逼，性能要么不达标硬上要么延期。

### 3. W3 设计 + 前端联调时引入了"客户原型"

第 3 周让一位**真实客户**（已签 NDA + 同意参与 beta）通过 Figma 看了原型。这位客户立刻指出："你们的 7 个数字里有一个其实我们不看（[特定指标]），但缺了我们最关心的 [另一个]。"

我们调整了 7 个数字的具体清单（PRD §2 没限定具体哪 7 个，预留了空间）。

### 4. 上线后 30 分钟监控 + 第 8 天复评

上线后部署人留岗 30 分钟监控（[`workflows/engineering/deployment_hygiene.md`](../../workflows/engineering/deployment_hygiene.md)）。
8 天后做了一次复评，对照 §5 验收：
- AC-1 ✅ / AC-2 ✅ / AC-3 ✅ (P95 = 280ms) / AC-4 ✅ / AC-5 ✅ / AC-6 ✅

## 做错的（已修复的）/ What Went Wrong (Fixed)

### W5：客户面发布说明差点漏红线 #2 违规词

工程同事起的发布说明初稿里有：

> 本次改版完成 PRD-0042 的所有验收标准。下个迭代我们会上线 [深度看板的 v2 增强]。

两处问题：
- "PRD-0042" — 内部 PRD 编号在客户面（红线 #2）
- "下个迭代" — 内部排期措辞暴露给客户（红线 #2）

红线 #2 三道闸时被 grep 关键词扫到 + 销售 lead 在品牌声音审阅时也指出。
改成：

> 创作者后台首页升级已完成。我们在准备一项深度看板的增强，敬请期待。

**教训**：每次写客户面文案都过 grep 闸，永远不要靠"我应该没漏吧"的感觉。

### W2 中段：工程 lead 顺手做了一处不在 PRD 范围的"小重构"

W2 期间，工程 lead 在改首页 API 时顺手把一段相邻的"通知 API"也重构了（觉得"反正都改这个文件"）。

代码评审时被发现 → 退回，要求拆 PR。理由：
- 这次重构没在 PRD 里登记
- 如果 dashboard 上线后通知出问题，rollback dashboard 会连带把通知 rollback 回原来的形态
- 违反 [`code_quality.md`](../../principles/subs/code_quality.md) §9（不要顺手重构）

工程 lead 拆了 PR，通知重构进入下一个独立的 PRD。

**教训**：哪怕你看到旁边有"明显能改进"的代码，**不要顺手**。开新 issue 让团队下次集中处理。

## 教学点 / Teaching Points

- **澄清问题前置**：开工前列不确定问题 = 节省后续返工
- **风险对策要执行**：写在 PRD §8 的风险对策不是"备份方案"，是"提前 trigger 的动作"
- **客户原型反馈**：在 PRD 没限定的"灵活段"上，让真实客户校准
- **客户面文案三道闸**：永远 grep 一遍，不靠"应该没漏吧"
- **不要顺手重构**：违反这条会拖累整个项目
- **五件套全过才能说"完工"**：每一项都不能跳
