# 案例 1: 创作者后台首页改版（2026 Q1）

> Creator Dashboard Redesign — 2026 Q1

- **主导职能**：产品
- **跨到的职能**：设计、工程、销售（合作客户预览）
- **持续时间**：6 周（2026-01-12 ~ 2026-02-23）
- **PRD 编号**：PRD-0042
- **状态**：已上线 / Shipped

---

## 一句话故事 / One-Liner

我们把客户后台的首页从"5 个不同模块各看各的"改成"一屏看到本周的核心数据 + 一键跳到深度看板"。这件事好的部分：流程把控住了。坏的部分：我们最初把"性能"目标写得太松，差点上线后才发现不达标——靠提早做的负载测试救了一把。

We reorganized the customer dashboard's home page from "5 separate modules each viewed independently" to "one screen for this week's core metrics + one-click into deeper dashboards". What went right: process discipline. What went wrong: we initially set the "performance" goal too loose; almost shipped without meeting it — saved by an early load test.
