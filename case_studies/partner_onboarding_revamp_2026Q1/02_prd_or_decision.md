# 02. 三份关键 ADR

> 重构的核心是 3 份独立但互相引用的 ADR。
> The revamp is anchored in 3 independent-but-cross-referenced ADRs.

---

## ADR-0023: 引入合作伙伴 3 级分级

```markdown
# ADR-0023: 合作伙伴 3 级分级

- 时间：2026-01-12
- 决策人：运营负责人 - L
- 状态：Accepted

## 背景
2025 年 18 家伙伴每家流程都按"重点客户级"对待。但实际上他们的合作模式 / 收入贡献 / 资源占用差异极大。

## 选项
- A. 不分级（现状）：所有伙伴一套流程
- B. 2 级（重点 / 普通）：简单但可能不够细
- C. 3 级（核心 / 战略 / 长尾）：更细但管理成本高

## 决策
选 C。理由：
- **业界标准**：B2B SaaS 同行普遍 3 级（参考行业访谈）
- **长期主义**：2 级在我们伙伴数过 30 后会显得粗，3 级可用到 100+ 规模
- **核心目标**：精准分级让我们对核心 / 战略伙伴投入更多，长尾伙伴用模板高效服务

## 后果
- 正面：每级有标准化的入驻 / 维护节奏
- 负面（取舍）：分级判定本身有摩擦——需要 ADR-0024 配合给"判定标准"
- 复评触发：合作伙伴数过 50 时 / 任何一级人数为 0 持续 6 个月时

## 关联
- ADR-0024（合同模板按级分）
- ADR-0025（入驻流程按级分）
```

## ADR-0024: 采用 3 套标准合同模板替代每家定制

```markdown
# ADR-0024: 标准合同模板（按 ADR-0023 分级分 3 套）

- 时间：2026-01-19
- 决策人：法务 + 运营负责人 - L
- 状态：Accepted

## 背景
2025 年法务审 18 份伙伴合同平均每份 5 工作日。模板化能把这个成本降到接近 0（除非有特殊条款）。

## 选项
- A. 完全定制（现状）：灵活但慢
- B. 1 套模板（强制所有伙伴用）：极快但伙伴可能不接受
- C. 3 套模板（按 ADR-0023 分级）+ 允许小幅修订：快 + 灵活

## 决策
选 C。理由：
- 业界标准：成熟 SaaS 公司多层模板 + 修订机制
- 长期主义：模板化让法务从"被动审 N 份"变成"主动维护 3 份"——容量上去了
- 核心目标：降低伙伴入驻摩擦 = 直接服务核心目标

## 后果
- 正面：法务审单时间从 5 天降到 1 天（小修订）/ 0 天（完全套模板）
- 负面（取舍）：失去对每家"独一无二条款"的灵活度。但 ADR-0023 的分级已经把伙伴大致分类，每类共性大于差异。
- 复评触发：累计 ≥ 3 家伙伴拒绝任何一套模板时 → 需要重新评估

## 关联
- ADR-0023
- ADR-0025
```

## ADR-0025: 入驻流程从"邮件接力"改成"工单 + 4 阶段 owner"

```markdown
# ADR-0025: 4 阶段入驻流程

- 时间：2026-01-26
- 决策人：运营负责人 - L
- 状态：Accepted

## 背景
现在的"邮件接力"流程没有明确 owner——销售把伙伴交给运营，运营卡在法务，法务等伙伴回签，伙伴回签后没人通知客户成功。结果整个流程没人对全程负责。

## 选项
- A. 加一个"伙伴成功经理"角色（新增 HC）
- B. 用工单系统 + 现有 4 角色按阶段轮流 owner（不新增 HC）
- C. 完全自动化（开发太重）

## 决策
选 B。理由：
- 业界标准：多数 SaaS 公司这么做（4 阶段 owner 是 B2B 标配）
- 长期主义：不新增 HC 让流程在公司规模 50→100→200 时仍然成立
- 核心目标：明确 owner 让流程不卡——直接服务伙伴入驻速度

4 阶段 + Owner：
1. 销售签意向 → 销售 owner（≤ 5 工作日）
2. 合同签订 → 法务 + 运营 co-owner（≤ 5 工作日）
3. 技术 / 商务 onboarding → 运营 owner（≤ 5 工作日）
4. 第一次合作落地 → 客户成功 owner（≤ 5 工作日）

总目标：≤ 20 工作日 = 4 周

## 后果
- 正面：每阶段有明确 owner，超期立刻可见
- 负面（取舍）：要求 4 个职能每周扫一次"我手里的工单"——增加 ~15 分钟 / 周
- 复评触发：连续 3 个伙伴超 4 周 / 任何阶段平均超 1 周

## 关联
- ADR-0023, ADR-0024
- 工单系统配置见 [runbook]（虚构链接）
```

## 教学点 / Teaching Points

- **3 份小 ADR > 1 份大 PRD**：分开决策让讨论聚焦
- **互相引用**：ADR-0023 → ADR-0024 → ADR-0025 形成决策链
- **每份 ADR 援引"长期主义 + 业界标准 + 核心目标"** —— 至高决策原则的真实应用
- **复评触发条件具体可观测**："过 50 家"、"任何一级人数为 0 持续 6 个月" 都是可数的
- **决策人是具体姓名**——不是"我们" / "团队"
