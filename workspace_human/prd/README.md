# workspace_human/prd/

> 人写的产品需求文档（PRD）。AI 只读（红线 #12）。
> Human-written PRDs. AI read-only (Red Line #12).

---

## 命名规则

```
PRD-NNNN_短标题.md
```

NNNN 是全公司统一的 4 位连续编号。下一个可用编号由产品负责人维护。

---

## 起新 PRD 的步骤

1. 复制 [`templates/prd/PRD-template.md`](../../templates/prd/PRD-template.md) 到本目录
2. 改文件名为 `PRD-NNNN_短标题.md`
3. 填写 §1-§9（AI 可以辅助起草，但你负责定稿和保存）
4. 提交评审 → §10 实施记录段后续由实施人填

---

## 子目录

- `compaction/` —— `COMPACT-NNNN` 反熵压缩提案。任何对仓库结构的拆分 / 归档动作必须先在此提一份 COMPACT 提案，等人授权再执行（红线 #11 + Chapter 0.2）
